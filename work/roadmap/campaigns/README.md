# Campaigns Registry

This directory stores only active or planned campaign charters/plans/evidence.

- Active execution campaign details should be mirrored in work/now/.
- Completed campaigns must be moved to `work/archive/campaigns/`.
- Legacy campaign assets are preserved as historical records in archive only.
- Lifecycle status buckets (`in progress / planned / done / parked`) are tracked in `work/now/campaign.md` and mirrored in `work/roadmap/backlog.md`.
- `agent/intructions/DTM-test/**` is reference-only input; execution tracking must be rebuilt and maintained here.

Current state:
- active execution follows the 2026-03-12 execution wave imported from owner-provided reference materials.
- campaign files here are the only working source of truth for plans/evidence inside the repo process layer.
