# Work Map

`work/` stores delivery process materials. System documentation lives in `docs/`.

## Sections
- `work/now/` - active campaign and execution checklist.
- `work/roadmap/` - planned campaigns and strategic queue.
- `work/archive/` - completed campaigns and historical process artifacts.

## Rules
- Campaign IDs: `CAM-<NAME>` with `P##` phases and `T###` tasks.
- Archive task history is summarized in campaign `closeout.md` files.
- Architecture and runtime behavior docs must stay in `docs/`.
